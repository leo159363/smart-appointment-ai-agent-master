# 领域迁移映射表

## 目标项目

| 原项目/领域概念 | 目标项目/领域概念 | 迁移说明 |
| --- | --- | --- |
| Smart Appointment AI Agent | AI 家教培训机构智能咨询与排课系统 | 项目定位从按摩门店预约系统迁移为家教培训机构的课程咨询、老师匹配、试听课预约和正式排课 Agent。 |
| 按摩门店 | 家教培训机构 | 业务主体从线下按摩服务门店改为提供一对一辅导、学科补弱、升学提分和学习习惯督促的培训机构。 |
| 智能预约系统 | 智能咨询与排课系统 | 对外能力强调课程咨询、学习需求分析、老师匹配、试听课预约、正式排课和排课冲突处理。 |

## 核心领域映射

| 原按摩场景 | 家教培训场景 | 迁移说明 |
| --- | --- | --- |
| 按摩门店 | 家教培训机构 | 首页、README、API 文档和系统介绍应体现家教机构，而不是门店服务。 |
| 技师 | 老师 | 对外文案统一为老师；内部 `technician` 命名可暂时保留，降低数据库、Repository 和路由风险。 |
| 技师性格/手法 | 老师教学风格/沟通方式 | 从手法力度、服务风格迁移为耐心程度、讲解方式、督促强度、互动方式。 |
| 服务项目 | 课程/学科 | 从全身推拿、肩颈推拿等迁移为小学数学、初中英语、高中物理、语文作文、一对一试听课等。 |
| 按摩预约 | 试听课预约/正式排课 | 预约流程从服务时段预约迁移为试听课安排、固定课次排课和老师时间匹配。 |
| 技师排班 | 老师可授课时间 | 可继续复用排班结构，但对外语义应改为老师课表、可授课时间、排课冲突。 |
| 用户偏好 | 学生年级、薄弱科目、学习目标、上课时间、老师风格偏好 | 偏好分析从顾客服务偏好迁移为学习需求和老师匹配条件。 |
| 技师专长 | 老师擅长科目、年级、教学风格 | 原 `strength` 字段可暂时承载老师擅长科目、适合年级、教学风格和提分方向。 |
| 顾客/客户 | 学生/家长 | 咨询语境以家长和学生为主，尤其是一对一辅导咨询。 |
| 门店地址 | 校区地址/上课地点 | 可包含线下校区、线上课程、上门家教和交通信息。 |
| 会员卡/充值 | 课程包/课时包 | 收费从会员充值迁移为试听价、课时包、续费、退费规则。 |
| 取消/改约政策 | 试听课改约/课程请假/退费规则 | 预约政策迁移为试听改约、正式课请假、补课和退费规则。 |

## Agent 映射

| 原 Agent | 目标 Agent | 迁移说明 |
| --- | --- | --- |
| 咨询 Agent | 课程咨询 Agent | 回答课程体系、适合年级、收费规则、老师介绍、试听课规则、退费规则和上课须知。 |
| 预约 Agent | 试听课预约/排课 Agent | 收集学生年级、学科、薄弱点、学习目标、可上课时间、老师风格偏好，并生成试听课预约或正式排课建议。 |
| 用户行为 Agent | 学习需求分析 Agent | 从咨询记录、预约记录和历史偏好中分析薄弱科目、学习目标、时间偏好、老师风格偏好和转化风险。 |
| 任务分类 Agent | 教育咨询任务分类 Agent | 区分课程咨询、试听课预约、正式排课、学习需求分析、价格咨询、无关问题等任务。 |

## 老师教学风格样例

| 风格类型 | 适合学生 | 对话/推荐语义 |
| --- | --- | --- |
| 耐心细致型 | 基础薄弱、容易紧张、需要慢节奏讲解的学生 | 强调拆步骤、慢讲解、及时反馈。 |
| 应试提分型 | 有明确考试目标、需要短期提分的学生 | 强调考点梳理、刷题策略、错题复盘。 |
| 互动鼓励型 | 主动性不足、需要增强兴趣和信心的学生 | 强调互动提问、正向反馈、学习兴趣。 |
| 严格督促型 | 自律性较弱、需要学习习惯管理的学生 | 强调作业检查、计划监督、阶段目标。 |
| 擅长基础补弱型 | 数学基础薄弱、物理概念不清、英语语法薄弱的学生 | 强调补基础、查漏补缺、知识框架。 |
| 擅长拔高竞赛型 | 成绩较好、需要拓展拔高或竞赛训练的学生 | 强调难题训练、思维拓展、竞赛专题。 |

## 学生需求样例

| 学生需求 | 可能匹配的课程/老师 | Agent 应收集的信息 |
| --- | --- | --- |
| 数学基础薄弱 | 数学一对一、基础补弱型老师 | 年级、当前成绩、薄弱章节、目标分数、可上课时间。 |
| 英语听说困难 | 英语听说专项、互动鼓励型老师 | 年级、教材版本、听说考试要求、口语基础。 |
| 作文不会写 | 语文作文专项、耐心细致型老师 | 年级、作文类型、常见问题、近期考试时间。 |
| 物理概念不清 | 初高中物理基础课、基础补弱型老师 | 年级、薄弱模块、教材版本、是否需要实验理解。 |
| 考前短期提分 | 应试提分课、应试提分型老师 | 考试时间、目标分数、当前分数、每天可投入时间。 |
| 需要老师督促学习习惯 | 学习规划课、严格督促型老师 | 作业完成情况、自律问题、家长期望、反馈频率。 |

## 保留内部命名的临时策略

本阶段不建议立即重命名数据库表、Repository、API 路由或核心变量，例如 `technician`、`appointment`、`service_type`、`project`。这些名称贯穿 FastAPI 路由、SQLite 模型、Repository、Service 和 Agent 流程，直接重命名容易破坏可运行性。

建议先完成对外文案、默认知识库、默认老师数据、Agent prompt 和测试用例迁移。等家教场景 MVP 跑通后，再评估是否做内部命名重构。
